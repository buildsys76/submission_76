Here is a Python script to generate this data:

```python
import random

# Building metadata and annual summary statistics (2016):
building_id = 272
primary_use = "Education"
square_feet = 4583
yearly_mean_kwh = 17.38018
yearly_min_kwh = 0.8
yearly_max_kwh = 74.32
summer_mean_kwh = 25.539513
winter_mean_kwh = 11.742976

# Generate electricity consumption data (kWh) at 1-hour intervals,
# starting on 01 January 2016, for exactly 3600 measurements.
start_date = "2016-01-01"
end_date = "2016-12-31"

with open("building_data.csv", "w") as f:
    start_timestamp = int(start_date.replace("-", "") + "000000")
    end_timestamp = int(end_date.replace("-", "") + "235959")

    for timestamp in range(start_timestamp, end_timestamp + 1):
        date = "{:%Y-%m-%d}".format(timestamp)
        hour = (timestamp % 86400) // 3600
        month = int(date[5:7])

        if month == 6 or month == 7 or month == 8:
            value = random.uniform(summer_mean_kwh * 0.9, summer_mean_kwh * 1.1)
        elif month == 12 or month == 1 or month == 2:
            value = random.uniform(winter_mean_kwh * 0.9, winter_mean_kwh * 1.1)
        else:
            value = random.uniform(yearly_mean_kwh * 0.9, yearly_mean_kwh * 1.1)

        value = max(yearly_min_kwh, min(value, yearly_max_kwh))

        f.write("{} {:04d}:{:02d},{}\n".format(date, hour, (timestamp % 86400) // 3600, round(value, 3)))
```

This script generates the electricity consumption data for each hour of the year, using a normal distribution around the yearly mean value. During summer months (June to August), it uses a normal distribution around the summer mean value; during winter months (December to February), it uses a normal distribution around the winter mean value. The values are capped at `yearly_min_kwh` and `yearly_max_kwh`, respectively.

The output is written to a file called `building_data.csv`.
